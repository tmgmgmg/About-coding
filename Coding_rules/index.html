基本事項

  • 1.見やすさ・編集のしやすさに努める
  • 2.正しい記述を心がける

1.見やすさ・編集のしやすさに努める

    • インデント

      インデントは半角スペース2文字分（コードエディタで設定可能）

    • css記述は1行にしない

      量が少なくても改行する

      また記述はブロックごとに固める

        .test {
    color: #fff;
  }

  • cssのプロパティをまとめる

    ショートハンドを使ってまとめてかける場合はまとめて書く

      【ショートハンドなし】
  .cell {
    margin-top: 112px;
    margin-right: 39px;
    margin-bottom: 0;
    margin-left: 0;
    padding-top: 26px;
    padding-right: 0;
    padding-bottom: 84px;
    padding-left: 0;
  }

  【ショートハンドあり】
  .cell {
    margin: 112px 39px 0 0;
    padding: 26px 0 84px;
  }

  • 色指定の16進数は短縮

    色の指定が3桁表記可能ならば短縮

        【改善前】
    .modal {
      color: #000000;
    }
    .content {
      color: #33bb00;
    }

    【改善後】
    .modal {
      color: #000;
    }
    .content {
      color: #3b0;
    }

  • インライン指定をしない

    htmlファイルへ直接styleの記述をせず、cssファイルへ記述する

  • !importantは極力使わない

    基本的には使用しない。!importantを使いたくなる時はcssが複雑化していることが考えられるため構造をシンプルにすることでデザインを実現できないか検討する

    詳細度を計算するよりも使用した方が簡単な場合もあるのでやむを得ない場合のみ使用する

  • cssで使用しないidやclassの記述を消す

    cssで使用しないidやclassの記述を残さず、提出時には削除する

  • グローバルセレクタには指定をしない

    打消しのためのcssが増えてしまうため汎用性の高い要素への指定は避ける

    例)　img、li、h1
    用いる場合は親要素にユニークな命名をする

  • クラスをチェーンしない

    以下のような指定は避ける

    .message.error{
}

.hedding.h1{
}

header.header{
}

  • cssファイルは、css整形ツールに入れてから提出する

    https://lab.syncer.jp/Tool/CSS-PrettyPrint/


    【設定】

    1. インデント　半角2
    2. セレクタ間の改行　有効
    3. ルール間の改行　有効
    4. 記号周りのスペース　有効

  • 担当箇所のコメントアウトは以下で統一

    /*———————————
サンプル
———————————*/

  • ベンダープレフィックスは以下サイトを通す

    https://autoprefixer.github.io

2.正しい記述を心がける

  • sectionタグ

    hタグがある時のみsectionタグを使用する。なければdivタグを用いる

  • hタグ（見出し）

    h1,h2,h3…と階層的に使い分ける

    h1タグはhtmlファイル1つにつき1回のみにとどめ、headerの会社logoイメージ挿入等に使う

  • tableタグ

    基本の形を崩さない

      <table>

      <tr>
        <th>table titleという意味。タイトルを記述する</th>
        <td>table descriptionという意味。詳細を記述する</td>
      </tr>

      <tr>
        <th>table titleという意味。タイトルを記述する</th>
        <td>table descriptionという意味。詳細を記述する</td>
      </tr>

  </table>

  • ulタグliタグ

    ulタグの中にliタグ以外を入れない

  • brタグ、hrタグ

    brタグを使った改行やhrタグを使った下線表示はしない

    同様にhtmlタグで見栄えに関するタグの使用はしない

    改行するときはspanタグを用いるか<div class="p">で囲む

  • id

    cssは基本的にidに記述しない

【EZgateコーディングルール】http://ezcodingrules.ezdev.jp/?EZ CordingRules

1.禁止事項

  • 「./」記述禁止

    以下のように書く。

    ./img/home-img01.jpg → img/home-img01.jpg

    ./はデフォルトで省略可能の記述。ついたままだと先方のCMSで検索した際にヒットしなくなるため禁止

  • ｐタグ禁止

    以下のように書く

    <div class="p">○○○○</div>

  • table利用制限

    tableタグは、
    ・お問い合わせ
    ・会社内容
    ・採用情報
    といった指定の場所以外は使用しない

  • CDN禁止

    jQueryなど使用の際はCDNではなく、ファイルを落としてきて、相対パスでリンクを貼る

    ※googleフォント、住所自動入力のAjaxZip3は例外

  • webフォント禁止

    指定のgoogleフォントのみ使用可

    googleフォントにないフォントの指示があった場合は、画像としてpsdから切り出す
    ※imgタグのalt属性には、切り出した文章を全て入力すること

  • 画像・ファイル名の_(アンダーバー)禁止

    ―(ハイフン)は可

  • Bootstrapのclass名に直接styleの指定禁止

    ・container
    ・row
    ・flex
    ・col-6
    などに直接styleの指定はNG

2.注意事項

  • alt記述をする

    飾り付けや雰囲気のみの画像以外は記入する

  • javascript

    htmlファイルに書かず、外部ファイルに記述する

  • 基本的に作るのはhtmlとcss

    PHPやjavascriptをつかって共通部分のheaderやfooterを読み込み・展開をしない

    共通部分はその都度htmlで記述する

  • 電話番号

    電話番号はスマホの時のみ発信できるようjsを用いて制御する

  • メールフォーム

    先方指定のコーディング方法があるため要確認

  • メールフォーム住所自動入力

    住所自動入力AjaxZip3のCDNは例外となっているためメールフォームのheadに記述する

    <script src="https://ajaxzip3.github.io/ajaxzip3.js">

  • 画像一覧表示

    カタログ（物件一覧、制作事例一覧等）は様々な縦横比の画像が挿入されるので、自動調節されるcssを使う

  • ページング

    先方共通のhtmlがあるためそれを使用しcssを合わせる

  • opacity:0;の時にクリックさせないようにする方法

    ハンバーガーやアニメーションで、opacityを用いて表示を隠す場合

    .example{
  opacity: 0;
  visibility:hidden;
  transition: all 0.5s;
}

.example:hover{
  opacity: 1;
  visibility:visible;
}

-----こちらでも可-----
.exsample2{
  opacity: 0;
  pointer-events: none;
}

.exsample2{
  opacity: 1;
  pointer-events: auto;
}

pointer₋iventsを使用する際はIE対策に要素をblockまたはinline₋blockにすること

  • IE読み込み時に、transitionが動作するバグへの対処

    以下コードを、各ファイルに記入

    ---HTML---
<body class="preload">

---CSS---
.preload * {
    -webkit-transition: none !important;
    -moz-transition: none !important;
    -ms-transition: none !important;
    -o-transition: none !important;
    transition: none !important;
}

---jQuery---
$(window).on('load' , function() {
  $("body").removeClass("preload");
});
